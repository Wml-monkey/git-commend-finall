1.git : 是一个用于版本控制的工具 2.gitHub： 是一个用于社交，查阅资料，查阅开源项目的网站，并且是一个远程仓库 3.git的三个区：工作区，暂存区，版本区（库） 4.git add ./name 从工作区添加到暂存区 5.git commit -m "注释" 从暂存区提交到版本库 （git commit -m "注释" --amend 如果有2个以上文件，一个提交到版本库了，另一个忘记提交，可以先将没提交的文件拉到暂存区，然后通过git commit -m “注释” --amend 撤销回来，最后自动一次性提交暂存区中的文件和撤销回来的版本形成一个新的版本，撤销回来的版本就销毁了） 6.git log 查看历史记录 7.git diff 从工作区看暂存区区别 8.git diff --cached（--staged）从暂存区看版本库的区别 9.git diff master 从工作区看版本库的区别 10.git reset HEAD "文件名字" 暂存区撤销到工作区 11.git checkout --"文件名字" 工作区代码还原暂存区 12.
